# Valdenmoor Chronicles

A medieval open-world text RPG powered by Claude as Game Master. Full free-form play — type anything.

## Stack

- **Frontend**: React + Vite (served as static files from Express)
- **Backend**: Node.js + Express (GM proxy, auth, saves)
- **Database**: PostgreSQL on Render
- **Auth**: JWT + bcrypt (manual user management — you add players)
- **AI**: Anthropic Claude API (server-side only — key never exposed to browser)

## Deploying to Render

### Step 1 — Push to GitHub

```bash
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/YOUR_USERNAME/valdenmoor-chronicles.git
git push -u origin main
```

### Step 2 — Create Render services

Go to [render.com](https://render.com) → **New** → **Blueprint** → connect your GitHub repo.

Render will read `render.yaml` and automatically create:
- A **Web Service** (your Express + React app)
- A **PostgreSQL database**

### Step 3 — Set your API key

In the Render dashboard → your web service → **Environment**:

| Key | Value |
|-----|-------|
| `ANTHROPIC_API_KEY` | `sk-ant-...` (your Anthropic key) |

`JWT_SECRET` is auto-generated by Render. `DATABASE_URL` is auto-connected from the database.

### Step 4 — Run the database schema

In Render dashboard → your PostgreSQL database → **Shell** (or use an external client):

```sql
-- Paste contents of server/schema.sql
-- This creates the users and saves tables and an initial admin account
```

The default admin password is `changeme` — **change it immediately** via the admin panel in the app.

### Step 5 — Deploy

Render auto-deploys on every `git push` to main. First deploy takes ~3 minutes.

Your app will be live at `https://valdenmoor-chronicles.onrender.com` (or similar).

## Adding Players

1. Log in as admin
2. Click **Admin** in the game header
3. Click **Add New Player** — enter their username and a temporary password
4. Tell them the URL and their credentials
5. They log in and start a new game — their save is automatically tied to their account

Each player has one save slot. You can wipe or reset saves from the admin panel.

## Local Development

```bash
# Install root deps
npm install

# Install client deps
cd client && npm install && cd ..

# Copy env file and fill in values
cp .env.example .env
# Edit .env with your ANTHROPIC_API_KEY, JWT_SECRET, DATABASE_URL

# Run server + client together
npm run dev
```

Server runs on `http://localhost:3001`  
Client runs on `http://localhost:5173` (proxies /api to server)

You need a local PostgreSQL instance. Create a database and run `server/schema.sql` in it.

## Architecture Notes

**The Anthropic API key never reaches the browser.** All GM calls go:

```
Browser → POST /api/gm → Express server → Anthropic API
                       ↑ API key lives here only
```

**The world bible and system prompt** live entirely in `server/index.js`. The browser never sees them — players can't read the GM's instructions in devtools.

**Saves** are stored in PostgreSQL per user. Each player has one row in the `saves` table, updated on every GM response.

**Auth** uses 30-day JWT tokens stored in localStorage. Tokens are validated on every `/api/*` request.

## Changing the Admin Password

Log in as admin → the game's "sign out" button → log back in → Admin panel → reset your own password is not available (admins can't edit themselves). Use psql directly:

```sql
-- Generate a bcrypt hash of your new password first (use any bcrypt tool)
UPDATE users SET password = '$2a$10$YOUR_HASH_HERE' WHERE username = 'admin';
```

Or: create a second admin account via psql, log in with that, then use the app UI.
